"我想研究今年年初白银暴涨的原因。此前只有基本常识，不知从何看起。"

自己找到了三篇不同范式的研报

我从白银领域的权威信息源中，找到了三篇覆盖不同研究范式的研报：国泰君安（期货展望，最详细的 45 页）、方正证券（供需结构 + 历史复盘，23 页）、银河证券（宏观周报视角，13 页）。

三篇加起来 81 页。如果自行阅读，起码需要 3 到 4 个小时。我把它们上传到系统，让 AI 帮我消化。
读
STEP 02
Read
"光是国泰那篇就 45 页。我需要快速消化核心要点，同时担心研报数据不准或逻辑有跳步。"

金字塔原则严格拆解每篇研报

两个核心需求：快速消化三篇研报的核心要点，以及严格拆解每个结论背后的逻辑链条，复核数据是否准确。

系统严格遵循 Minto 金字塔原理拆解逻辑，用 IB Color Code 标注数据可信度（蓝=引用、绿=假设、黑=计算），补充逻辑跳步让新入门读者也能看懂。

支持在原文复刻（忠实原报告推导逻辑）和逻辑重组（对原逻辑进行再度重组优化）两种视图间切换。

审
STEP 03
Verify
"报告里的数据靠谱吗？那些'因为 A 所以 B'的推导，真的站得住脚吗？"

多 Agent 并发：既审数据，也审逻辑

审数据：每个数据节点分配独立的验证 Agent，溯源到一手来源（Silver Institute、CME、LBMA），比对数据是否有误。

审逻辑：每条因果推理独立检验，判断逻辑推导是否严密，有没有关键跳步。

全部 Agent 并发执行，大幅加速审计效率。

沉
STEP 04
Distill
"验证完了这么多数据点，哪些是正确的核心判断？哪些 Source 以后还用得上？"

沉淀正确的核心判断与可追踪的知识资产

从验证结果中沉淀两类资产：

核心判断：经过验证的、对投资主线有决定性贡献的关键事实。红色剔除，黄色择优，绿色中只保留对故事有贡献的。

知识资产库：从不同 Source 获取的定量数据，按分类整理成结构化表格，每条都有可追踪的一手来源 URL。这不是一次性产物，而是未来持续追踪和深入研究的起点。
合
STEP 05
Merge
"三篇报告加起来有四五十个零散维度。国泰从期货角度看，方正从供需看，银河从宏观看。我需要一个大一统的视角。"

从零散维度中，合成一条逻辑主线

让 AI 结合金字塔原理，从四五十个零散维度中寻找一个"大一统"视角，把逻辑理清楚。按 WHY（为什么有结构性问题）、HOW（问题怎么变成危机）、CONTEXT（宏观大背景）、RISK（什么情况会反转）组织。

每个结论都标明已验证还是未验证。像是你自己花一整天读完 81 页后，在脑子里形成的那条主线。

"主线懂了。但白银的故事还在继续，接下来具体该盯哪些数据？"

按逻辑主线，AI 告诉你该追踪什么

根据"合"出的数据线，AI 确定后续应关注的具体数据，按供需面（供需平衡、缺口趋势、ETF 持仓）、现货面（COMEX 库存、LBMA 库存、Lease Rate）、宏观面（FedWatch、金银比、DXY）三大框架组织。

支持自定义看板追踪。追踪过程中复用"沉"步沉淀的 Source 和逻辑框架，让你持续判断主线还成不成立、什么时候该修正判断。
