Demo · 白银投资主线 Journey

假设你是一个普通研究员，想搞懂今年白银为什么涨。你之前没研究过有色金属，但下载了几篇券商研报。 以下是系统帮你完成的整个旅程：

01
搜
"我想研究白银上涨的原因，但不知道该看哪些报告。"

系统从权威源搜索了 3 篇不同视角的研报

国泰君安（期货展望，45页）、方正证券（供需+历史，23页）、银河证券（宏观视角，13页）。三篇覆盖不同范式，共 81 页。

02
读
"45 页的报告太长了，我没时间从头读。"

系统将每篇研报拆解为金字塔结构

用 Minto 金字塔原则提取逻辑框架，标注 IB Color Code（蓝=引用、绿=假设、黑=计算），补充常识跳步。你可以在原文复刻和逻辑重组之间切换。

△ 国泰君安 · 金字塔 △ 银河证券 · 金字塔 △ 方正证券 · 金字塔
03
审
"报告里的数据靠谱吗？逻辑链条能跑通吗？"

19 个子 Agent 并发验证每个数据节点和因果边

每个数据点独立溯源到一手来源（Silver Institute、CME、LBMA），每条因果推理独立检验。三色染色：绿=验证通过、黄=待验证、红=有冲突。点击任意节点查看完整验证底稿。

✓ 国泰君安 · 验证图 ✓ 银河证券 · 验证图 ✓ 方正证券 · 验证图
04
沉
"这么多数据点，哪些是真正重要的？"

沉淀关键事实与核心 Source，构建知识资产

系统从验证通过的数据中筛选出对投资主线真正有贡献的关键事实，记录每个数据的一手来源 URL，形成可追踪的知识资产库。无关但正确的数据降级为佐证。

05
合
"三篇报告各说各的，到底该信谁？"

三条逻辑线合并为一条完整的投资主线

用 Storyteller 方法先确定故事（"白银面临结构性供需缺口，现货机制正将其转化为逼空风险"），再按 WHY/HOW/CONTEXT/RISK 搭建共识树。包含完整供需数据链：供给 1015 Moz - 需求 1164 Moz = 缺口 -149 Moz。

◆ 合并投资主线
06
追
"主线我懂了，但后面该盯什么指标？"

从主线中导出核心指标，生成定制化追踪看板

按供需面/现货面/宏观面三大框架，追踪 COMEX 库存、Lease Rate、FedWatch 等核心指标。每个指标附 YTD 趋势分析和近期异动预警。数据自动从一手来源更新。
