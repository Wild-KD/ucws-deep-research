# 追：投资主线 + Source 注册表 → 定制化追踪看板

你收到两个输入：
1. **合并共识树**（"合"技能的产出）— 告诉你该追踪什么逻辑
2. **Source 注册表**（"沉·整"技能的产出）— 告诉你去哪里拉数据

根据投资主线确定需要追踪的核心指标，从 Source 注册表中已登记的一手来源拉取最新数据，生成可持续监控的看板。

---

## 本技能不做的事

- 不重新分析投资逻辑（那是"合"技能）
- 不使用二手数据（不用研报里的图表，从 Source 注册表的一手来源拉）
- 不生成投资建议
- 不自己发明该追踪什么——追踪对象从投资主线推导

---

## 第 1 步：提取监控指标

从合并树的每个主干中，提取可量化、可持续追踪的指标。

### 1.1 按三大框架分类

| 框架 | 包含什么 | 典型指标 |
|------|---------|---------|
| **供需面** | 基本面数据 | 供需平衡表、缺口趋势、ETF持仓 |
| **现货面** | 市场微观结构 | 交易所库存、Lease Rate、持仓数据 |
| **宏观面** | 外部环境 | 利率预期、汇率、跨资产比价 |

### 1.2 每个框架选 3 个核心指标

→ 总共 9 个左右。不要贪多。
→ 选择标准：该指标的变化是否直接影响合并树的核心论点？

### 1.3 记录每个指标的元信息

| 字段 | 必填 |
|------|------|
| 指标名称 | ✅ |
| 数据口径说明 | ✅ |
| 一手数据源（机构名） | ✅ |
| 更新频率（日/周/月/季/年） | ✅ |
| 数据源 URL | ✅ |
| 单位 | ✅ |

---

## 第 2 步：从 Source 注册表拉取最新数据

**优先使用"沉·整"步已经登记的 Source。** 不要重新搜索——沉步已经验证过 URL、登记过数据维度。

### 2.1 数据拉取顺序

1. **先查 Source 注册表**：该指标对应的 Source 是否已登记？URL 是否可用？
   → 已登记且可用 → 直接 `web_fetch` 该 URL 拉最新数据
   → 已登记但 URL 失效 → 用 `web_search` 搜该 Source 的新 URL
2. **沉·探发现的新数据**：如果"沉·探"推荐了某个数据维度纳入追踪，优先使用
3. **补充搜索**：只有 Source 注册表没覆盖的指标，才从头搜索

### 2.2 数据源优先级（补充搜索时）

1. 官方机构（Silver Institute、CME、LBMA、ICE、各国央行）
2. Bloomberg / Reuters
3. 行业协会

→ **严禁使用二手数据。** 不要直接用研报里的图表数据。

### 2.3 搜索策略（补充搜索时）

对每个指标：
1. 搜 `{指标名} {年份} {来源机构}` 找最新值
2. 搜 `{指标名} historical data` 找时间序列
3. 需要至少 6 个月的历史数据做趋势图

### 2.3 记录数据

```json
{
  "indicator": "COMEX白银库存",
  "current_value": 340,
  "unit": "Moz",
  "as_of": "2026-05-29",
  "source": "CME Group",
  "source_url": "https://...",
  "frequency": "日度",
  "history": [
    {"date": "2026-01", "value": 440},
    {"date": "2026-02", "value": 410},
    ...
  ]
}
```

## 第 3 步：AI 分析

对每个指标，写两段简短分析：

### 3.1 YTD 趋势分析

回答："年初到现在，这个指标的大趋势是什么？对论点有什么含义？"
- 1-2 句话
- 包含方向（上/下/稳）和幅度
- 联系到合并树的论点

### 3.2 近期异动分析

回答："最近有没有异常变动？如果有，意味着什么？"

→ 有异动：描述异动 + 解释可能原因 + 标记为警示
→ 没有异动：可以省略或简单写"近期无显著异动"

**什么算异动？**
- 单日/单周变动超过历史月均的 2 倍
- 突破近 6 个月区间的上沿或下沿
- 和其他指标出现背离（如现货紧张但 ETF 在减仓）

## 第 4 步：构建看板布局

```
┌─────────────────────────────────────┐
│ 标的价格走势（全宽，Chart.js折线图）   │
├─────────────────────────────────────┤
│ Summary KPIs（一行，6个核心数字）      │
├─────────────────────────────────────┤
│ 供需面：3列                          │
│ [指标1] [指标2] [指标3]              │
│ 每列：标题+口径+图表+YTD+异动         │
├─────────────────────────────────────┤
│ 现货面：3列                          │
│ [指标4] [指标5] [指标6]              │
├─────────────────────────────────────┤
│ 宏观面：3列                          │
│ [指标7] [指标8] [指标9]              │
└─────────────────────────────────────┘
```

## 第 5 步：图表规范

所有图表使用 Chart.js（CDN引入）。

### 折线图要求
- 必须有坐标轴标签（X轴=时间，Y轴=单位）
- Hover 显示具体数值 + 单位
- 线条 pointRadius:0（平时不显示点），pointHoverRadius:4（hover时显示）
- 线宽 1.5px，tension:0.3
- 颜色：每个框架用一个色调，不要太花

### 每个图表卡片包含
1. 标题 + 更新频率标签（右上角小字）
2. 数据口径说明（标题下一行灰色小字）
3. Chart.js 图表
4. YTD 趋势分析（图表下方）
5. 近期异动分析（如果有，用警示样式）

---

## 第 6 步：生成 HTML

### 技术要求
- 单文件 HTML，所有数据内联
- Chart.js 通过 CDN 引入
- 全屏布局，CSS Grid
- 设计风格对齐 AskMBB（深色背景 #141417，金色 accent #d4a76a）
- 响应式：大屏 3 列，小屏 1 列

### 数据声明
页脚注明：
- "Demo 数据基于联网搜索近似值"
- "实际产品应接入 Bloomberg Terminal / CME DataMine / LBMA API"
- 每个指标的更新频率

---

## 第 7 步：质检

| 检查项 | 不通过则 |
|--------|---------|
| 所有图表都渲染了？ | 检查 Chart.js CDN 和 canvas ID |
| Hover 有数据显示？ | 检查 tooltip 配置 |
| 数据是最新的（不是半年前的）？ | 重新搜索 |
| 每个图表都标了口径和来源？ | 补上 |
| 供需平衡表的数字能验算？ | 供给-需求=缺口 |
| 中文标签显示正常？ | charset=UTF-8 |

使用 `html_writer` 将看板 HTML 写入指定路径。
