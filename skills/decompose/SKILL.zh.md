# 读：研报 → 金字塔结构

你收到一篇研报（PDF 或文本）。产出两个东西：
1. **原文金字塔** — 忠实还原报告自身的逻辑结构
2. **重组金字塔** — 按因果逻辑重建，补跳步，标颜色，让人能把这个故事讲给别人听

---

## 本技能不做的事

- 不验证数据真伪（那是"审"技能）
- 不合并多篇报告（那是"合"技能）
- 不添加报告外的任何数据。每个数字必须来自报告原文。

---

## 第 1 步：加载报告

1. 用 `pdf_reader` 读取完整报告
2. 记录元数据：标题、机构、发布日期、总页数
3. 通读一遍，用一句话回答：这篇报告的核心论点是什么？

产出：一句话论点 + 元数据。

---

## 第 2 步：原文复刻

保留报告原来的逻辑结构，不改写、不重组。

对每个章节：
- 保留报告原始标题文字
- 提取该章节下的关键数据点和论断
- 每个数据点标注**页码**

→ 这就是**原文金字塔**。保证它还原了原报告的推导逻辑，让用户能看到"报告作者自己是怎么想的"。

---

## 第 3 步：找到主干

**忘掉报告自身的章节结构。** 问自己：如果我要把这篇报告的核心观点讲给一个完全不了解这个领域的人听，我需要讲几件事？

### 3.1 提取所有论点

先把报告中所有的论点、判断、数据驱动的结论全部列出来，不分层级，不做筛选。这是原材料。

### 3.2 聚类为主干

从这些零散论点中找规律：哪些在说同一件事的不同侧面？

**心理测试：** 假设你在给老板做 15 分钟的汇报。你会把这些论点分成几个大块来讲？每个大块就是一个主干。

根据报告类型参考：

| 报告类型 | 典型主干 |
|---------|---------|
| 大宗商品展望 | 供给 / 需求 / 宏观 / 风险 |
| 公司分析 | 商业模式 / 财务 / 估值 / 催化剂 |
| 行业综述 | 市场规模 / 竞争格局 / 增长驱动 / 风险 |
| 宏观研究 | 现状 / 政策 / 传导机制 / 前瞻 |

### 3.3 检查主干之间的关系

**关键判断：** 某两个主干之间有因果关系吗？
→ 有（A导致B）：合并为一个主干，A→B 是该主干的内部逻辑链
→ 没有（A和B独立）：保持为两个并列主干

每篇报告应有 **2-4 个主干**。超过 4 个说明有些是分论点不是主干。

---

## 第 4 步：写 Minto 标题

**这一步决定了别人能不能只看标题就理解你的故事。**

每个层级的标题必须是**方向性结论**，不是分类标签。

### 测试方法

遮住所有正文，只读标题，从上到下。问自己：
- 我能理解完整论证吗？→ 能 → 标题合格
- 我只看到一堆分类名？→ 标题是标签不是结论，必须改

### 如何转换

| ❌ 分类标签 | ✅ 结论先行 |
|-----------|-----------|
| 供给端 | 供给短期收缩，伴生矿制约增产空间 |
| 利率 | 降息延续降低贵金属持有的机会成本 |
| 需求展望 | 需求不会断崖，光伏转型对冲传统下滑 |
| 历史对比 | 当前格局类似 2010-2011 逼空前夜 |

**规则：**
- 标题必须包含方向（涨/跌/稳/风险）
- 标题必须包含机制或原因（"因为…"、"由于…"、"在…驱动下"）
- 如果写不出结论 → 这个章节可能需要拆分或合并

---

## 第 5 步：MECE 拆分（核心步骤）

**这是整个技能最关键的步骤。**

目标不只是"互斥完备"。拆完之后，**每一层**都必须同时满足三个标准：

1. **MECE 完备** — 不遗漏、不重叠
2. **逻辑相关性** — 同层的分支之间有清晰的逻辑先后或并列关系
3. **Storytelling 自检** — 按这个顺序能给别人讲一个连贯的故事

这三个标准在金字塔的**每一层**都要检查，不只是顶层。

### 5.1 第一步：提取所有论据

把报告中属于该主干的所有论断、数据点、推理全部列出来。先不筛选，先穷举。

### 5.2 第二步：识别推理线

**在这堆论据中，有几条独立的推理线（reasoning lines）？**

一条推理线 = 一个完整的"因为A所以B"链条。

举例（白银供给主干下）：
- 推理线1：矿产银受伴生矿制约 → 即使银价涨也难增产 → 供给端硬约束
- 推理线2：再生银回收量稳定 → 补不上矿产缺口 → 供给弹性低
- 推理线3：部分矿企在减产 → 短期供给进一步收缩

每条推理线就是一个分论点。

### 5.3 第三步：三重检查（每一层都做）

**检查一：MECE 完备**

互斥性：
→ 对每一对推理线，问：它们有交叉吗？同一个论据可能属于两条线吗？
→ 有交叉：合并它们，或重新划边界
→ 没有交叉：保持独立

常见违规：
- "短期因素" vs "长期因素"，但有些数据两边都沾 → 改为按机制分
- "价格因素" vs "成本因素"，但成本就是价格的一部分 → 合并
- 因果链上的前后事件被当成并列 → 合并为一条推理线

完备性：
→ 报告中有没有重要论据不属于任何一条推理线？
→ 有：增加或扩大 → 没有：覆盖完整

**检查二：逻辑相关性**

同一层的分支之间，必须有清晰的逻辑关系：

→ 它们之间是**并列**关系吗？（同等重要的不同侧面）
→ 它们之间是**递进**关系吗？（A是前提，B是推论）
→ 它们之间是**对比**关系吗？（正面 vs 反面）
→ 它们之间是**因果**关系吗？→ 如果是，不应该并列，应该合并为一条链

**即使是并列的分支，也有逻辑上的最优顺序：**
- 如果有前提→推论关系 → 前提在前
- 如果是不同维度 → 最核心的维度在前
- 如果是正反对比 → 主要观点在前，反驳在后

**检查三：Storytelling 自检**

问自己：如果我按照这个结构给别人做 presentation：
→ 按这个顺序讲下来，听众能跟上吗？
→ 讲到某个点时，听众会不会问"为什么突然讲这个"？→ 如果会，顺序有问题
→ 讲完所有分支后，听众能自然地得出父节点的结论吗？→ 如果不能，分支不完备或结论太跳跃

### 5.4 强制约束

- **每层最多 4 个分论点。** 超过 4 个说明有两条推理线其实是同一件事，合并。
- **并列对称。** 同一父节点下的分论点必须是同类（都是原因、都是结果、都是参与方），不能混搭。
- **因果链合并。** A 导致 B → 它们不是并列分论点，而是一条推理线的内部结构。

### 5.5 递归

对每个分论点重复第 5.2-5.4 步，直到到达叶节点（单一数据点或声明）。

**每一层都做三重检查**，不只是顶层。叶节点的上一层也要满足 MECE + 逻辑相关性 + Storytelling。

---

## 第 6 步：写叶节点

每个叶节点格式：

```
[IB颜色加粗] 一句话新闻导语风格（5W1H）
*(来源：报告名，第XX页)*
```

### IB Color Code（文字颜色——只有三色）

| 颜色 | 色值 | 什么时候用 | 举例 |
|-----|------|----------|------|
| **蓝** | #005EB8 | 引用外部数据源 | "LBMA 金库持有量降至 22,858 吨（LBMA月报）" |
| **绿** | #008800 | 作者的假设、预测 | "我们预计光伏用银需求年增 15% 至 2028" |
| **黑** | #000000 | 计算结果、更新说明、结构性文字 | "缺口 = 供给1015 - 需求1164 = -149 Moz" |

**每个数据点的判断：**
→ 这个数字来自报告引用的外部源？→ 蓝色
→ 这是报告作者自己的预测/假设？→ 绿色
→ 这是从其他数字算出来的？→ 黑色
→ 不确定？→ 默认蓝色（外部引用），并标注不确定

### 常识节点

当报告的逻辑有跳步、非专业读者会看不懂时，插入常识节点：

```
[常识] 当交易所仓库库存低于可交割合约量时，空头被迫回补买入（"逼空"），
导致价格在基本面之上进一步飙升。
*(来源：大宗商品市场标准机制)*
```

- 灰色文字，标签 `[常识]`
- 仍然要标来源（即使是"市场标准机制"或"教科书定义"）
- 可视化时可隐藏/显示

---

## 第 7 步：供需数据链（如适用）

如果报告涉及有供需关系的商品/行业，**必须**构建完整数字链：

```
供给：
  矿产银：XXX Moz
  + 再生银：XXX Moz
  + 其他：XXX Moz
  = 总供给：XXX Moz

需求：
  工业需求：XXX Moz
    其中光伏：XXX Moz
  + 珠宝：XXX Moz
  + 投资：XXX Moz
  = 总需求：XXX Moz

平衡：总供给 - 总需求 = 缺口/盈余 XXX Moz
```

**规则：**
- 必须有绝对数字，不接受只有百分比
- 读者必须能自己验算：供给 - 需求 = 报告说的缺口？
- 如果报告有多种口径（含/不含 ETF），并列展示
- 如果报告给区间，写 "XXX-YYY Moz"

---

## 第 8 步：反幻觉检查

**必做。** 逐条检查重组金字塔中的每个数据点：

| 检查项 | 不通过则 |
|--------|---------|
| 这个数字在报告原文中明确出现了吗？ | 没有 → 删掉 |
| 单位正确吗（Moz、吨、%、美元）？ | 不对 → 按报告原文修正 |
| 我做了报告里没有的计算或换算吗？ | 有 → 删掉计算，只保留报告原有数字 |
| 我翻译了引用并把译文当作原文呈现了吗？ | 有 → 改回原文语言 |
| 每个数据点都有页码吗？ | 没有 → 找到页码或删掉 |

**已知幻觉模式（特别警惕）：**
- 换算报告没换算的单位（如"42640吨"→"覆盖1.2个月"，但报告从未这么说）
- 拼接不同表格/章节的数字做计算
- 把中文报告的引用翻译成英文，当作直接引用呈现
- 和报告不同的舍入方式

**发现幻觉时：** 直接删除，不要试图搜索正确数字。只用报告明确写的内容。

---

## 第 9 步：逆向复核

只读重组金字塔的标题，从上到下。

问三个问题：
1. **只看标题能理解完整论证吗？** → 不能 → 某个标题是标签不是结论，修改它
2. **相邻两个标题之间有逻辑跳步吗？** → 有 → 插入常识节点
3. **故事自然流畅、不需要回头看吗？** → 不是 → 调整顺序

最后做一次 presentation 检验：用这个结构能给一个不懂这个领域的人讲清楚吗？
