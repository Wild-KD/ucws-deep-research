# 沉·探：Source 注册表 → 数据地图

你收到"沉·整"技能产出的 Source 注册表。对其中每个一手数据源进行深挖，mapping 它的完整数据版图，发现研报中没有提到但对投资研究有价值的数据。

---

## 本技能不做的事

- 不验证数据准确性（那是"审"技能）
- 不构建注册表（那是"沉·整"技能）
- 不判断投资主线（那是"合"技能）
- 不下载付费数据（只 mapping 免费可获取的部分）

---

## 第 1 步：筛选深挖目标

从 Source 注册表中，筛选值得深挖的 Source：

| 条件 | → 是否深挖 |
|------|-----------|
| 一手数据源 + URL 可用 + 权威/可靠 | ✅ 必须深挖 |
| 官方统计 + URL 可用 | ✅ 值得深挖 |
| 二手引用 | ❌ 跳过（去找它引用的一手源） |
| URL 不可用或付费墙 | ❌ 跳过（标记为"需替代方案"） |

---

## 第 2 步：Mapping 每个 Source

对每个深挖目标，执行以下探索：

### 2.1 访问主页，理解网站结构

用 `web_fetch` 打开 Source 的主 URL。**像一个研究员第一次打开这个网站一样，把它的结构摸清楚：**

- 这个机构/网站是做什么的？它的核心业务是什么？
- 导航栏有哪些板块？（Data / Research / Publications / Statistics / Downloads）
- 有没有专门的数据中心/数据库入口？
- 有没有 API 或数据下载页面？
- 有没有免费 vs 付费的分区？

### 2.2 逐层深入，打开关键子页面

不要只看首页。**点进去。**

→ 找到"Data"或"Statistics"入口 → `web_fetch` 打开它
→ 找到"Publications"或"Reports" → 看看有哪些定期发布物
→ 找到"Downloads"或"API" → 记录数据获取方式
→ 如果有搜索功能 → 用 `web_search` 搜 `site:source-domain.com {相关关键词}`

**目标是画出这个网站的数据地图：它有什么、在哪里、怎么拿到。**

### 2.3 逐个数据维度登记

对发现的每个数据维度，记录：

```
维度名称: COMEX白银库存
数据类型: 时间序列
更新频率: 日度
时间范围: 2000-至今
获取方式: 免费网页 / API / 需注册 / 付费
URL: https://...
格式: 网页表格 / CSV下载 / JSON API
与投资主线的关系: 直接相关（现货面核心指标）
```

### 2.4 判断与投资主线的关系

对每个发现的数据维度，问：

→ 这个数据和我们的投资主线有关系吗？
  → **直接相关**：是投资主线中某个论点的核心数据（如供需缺口）
  → **间接相关**：能辅助判断但不是核心（如白银在某个应用领域的消费量）
  → **潜在价值**：目前主线没用到，但如果主线修正可能会用到
  → **无关**：和白银投资完全无关，跳过

---

## 第 3 步：发现研报遗漏的数据

对比研报实际引用的数据 vs Source 完整数据版图，找出：

| 类型 | 举例 |
|------|------|
| **研报引用了但没充分利用的** | 报告引用了WSS的年度数据，但WSS其实还有季度数据 |
| **研报完全没提到的** | Silver Institute有月度ETF持仓追踪，但报告没用 |
| **同一Source的不同粒度** | 报告用了年度数据，但月度/日度数据也存在 |
| **同一Source的历史回溯** | 报告只看了近两年，但数据可以回溯10年 |

---

## 第 4 步：输出数据地图

### 4.1 每个 Source 一张地图

```
┌─────────────────────────────────────────┐
│ Silver Institute                         │
│ silverinstitute.org                      │
│                                         │
│ ┌─ 已用 ─────────────────────────┐      │
│ │ · 年度供需平衡 (年度, 2015-26E) │      │
│ │ · 矿产银产量 (年度)             │      │
│ │ · 光伏用银 (年度)               │      │
│ └────────────────────────────────┘      │
│                                         │
│ ┌─ 未用但直接相关 ──────────────┐       │
│ │ · 月度ETF持仓 (月度, 免费)     │       │
│ │ · 银价历史 (日度, 免费)        │       │
│ └────────────────────────────────┘      │
│                                         │
│ ┌─ 潜在价值 ──────────────────┐         │
│ │ · 再生银来源分析 (年度, 付费)  │         │
│ │ · 区域消费分布 (年度, 付费)    │         │
│ └────────────────────────────────┘      │
└─────────────────────────────────────────┘
```

### 4.2 推荐纳入追踪的数据

从"未用但直接相关"中，推荐哪些应该纳入"追"步的看板追踪：

→ 更新频率够高（至少月度）？
→ 免费可获取？
→ 对判断投资主线是否仍然成立有帮助？

全部满足 → 推荐纳入追踪

---

## 第 5 步：作为数据资产沉淀

探索完成后，这些信息不是用完就扔的——它们是**可复用的数据资产**。

### 5.1 构建该领域的"数据源手册"

把所有探索结果整理成一份结构化的手册，未来研究同一领域时可以直接复用：

- **去哪找什么数据** — 每个 Source 能提供什么，更新频率，获取方式
- **哪些是免费的** — 免费数据的完整清单，减少对付费终端的依赖
- **数据之间的关系** — 比如 Silver Institute 的年度数据 + CME 的日度数据 = 年度框架 + 日度追踪的组合

### 5.2 标记"追"步可直接使用的数据

对推荐纳入追踪的数据，记录足够详细的获取路径，让"追"步可以直接拉数据：

```
数据: COMEX白银注册库存
Source: CME Group
URL: https://www.cmegroup.com/...（具体页面，不是首页）
获取方式: 网页表格，每日更新
数据格式: 在线表格，可手动抓取
备注: 注册库存和合格库存分开展示，追踪时两个都要看
```

### 5.3 记录探索过程中发现的意外收获

探索过程中可能发现一些意料之外的有价值信息：
- 某个 Source 提供了完全不同角度的数据（如环保法规对银矿的影响）
- 某个 Source 有历史数据可以做回测
- 某个 Source 的数据和另一个 Source 有口径差异，需要注意

这些也要记录下来，作为知识资产的一部分。
