# 搜：给定标的 → 搜集该领域的高质量研究材料

你收到一个投资标的或研究问题（可能附带用户上传的 PDF）。你的工作是尽可能全面地搜集与该问题相关的高质量材料，供后续步骤使用。

---

## 本技能不做的事

- 不拆解报告内容（那是"读"技能）
- 不验证数据（那是"审"技能）
- 不决定最终使用哪几篇——搜集尽可能多，让用户自己选

---

## 第 1 步：理解用户的问题

### 1.1 拆解研究问题

用户给的可能是一个标的（"白银"）、一个问题（"今年白银为什么暴涨"）、或一个方向（"有色金属赛道"）。

先把它拆成可搜索的子问题：
- 这个标的属于什么领域？（大宗商品/个股/行业/宏观）
- 核心问题是什么？（价格驱动/供需结构/估值/政策变化）
- 时间范围？（近期事件/长期趋势/历史复盘）

### 1.2 如果用户上传了 PDF

用户上传的报告是最重要的信号——它告诉你用户的**研究偏好**：

**分析上传报告的特征：**
- 机构类型（券商/期货公司/买方/咨询/媒体）
- 研究深度（深度报告/周报/快评）
- 分析视角（供需基本面/技术面/宏观/产业链）
- 数据密度（大量图表 vs 定性为主）
- 语言（中文/英文/双语）

**据此推断用户偏好：**
→ 用户上传了一篇 45 页的期货公司深度报告 → 偏好深度、数据密集型研究
→ 用户上传了一篇 Bloomberg 短评 → 偏好简洁、观点鲜明的评论
→ 用户上传了中英文各一篇 → 希望跨语言视角

**补充搜索策略：**
- 搜索与上传报告**同类但不同视角**的材料（同一问题，不同机构/不同范式）
- 搜索上传报告**引用的 Source**的原始文档（顺藤摸瓜）
- 搜索上传报告**作者/机构**的其他相关研究（跟踪该分析师）
- 如果上传的都是卖方研报 → 主动补充买方视角和媒体评论

---

## 第 2 步：预研——找到该领域最好的分析师和来源

**在正式搜索之前，先花一轮搜索搞清楚：谁是这个领域最权威的声音？**

### 2.1 搜索该领域的权威分析师

搜索 `{标的} best analyst` / `{标的} 知名分析师` / `{标的} top research`

目标是识别：
- 该领域被引用最多的分析师/团队
- 长期跟踪该标的的专家
- 有独特视角的独立研究者

### 2.2 搜索该领域的权威信息源

不同领域有不同的核心信息源：

| 领域 | 权威信息源 |
|------|----------|
| 贵金属/大宗商品 | Silver Institute, World Gold Council, LBMA, CME, Bloomberg Commodities |
| 个股/美股 | SEC Filings, Seeking Alpha, Morningstar, 投行研报 |
| A股 | Wind, 同花顺, 券商研报, 巨潮资讯 |
| 行业/赛道 | 行业白皮书, 咨询公司(麦肯锡/BCG), Statista, IBIS World |
| 宏观 | Fed, ECB, BIS, IMF, 央行工作论文 |

### 2.3 记录预研结果

列出：
- 3-5 个该领域最权威的分析师/团队
- 3-5 个该领域最核心的信息源
- 该问题近期是否有重大事件（影响搜索时间窗口）

---

## 第 3 步：多轮搜索

**不要只搜一轮。搜索策略应该不断演化。**

### 3.1 第一轮：广撒网

构造多组搜索词，覆盖不同角度：

```
{标的} 研报 {年份}
{标的} outlook {year}
{标的} 供需 分析 / {标的} supply demand
{标的} 深度报告 / {标的} research report
{标的} {关键事件} （如"白银 逼空"、"白银 光伏需求"）
```

中英文都搜。每组取前 5-10 条结果。

### 3.2 第二轮：追踪预研发现的权威来源

用第 2 步发现的分析师和信息源名字做定向搜索：

```
{分析师名} {标的} {年份}
site:{权威信息源域名} {标的}
{机构名} {标的} report
```

### 3.3 第三轮：顺藤摸瓜

从已找到的报告中，提取它们引用的来源，搜索这些来源的原始文档：
→ 报告引用了 "Silver Institute World Silver Survey 2026" → 搜索该报告原文
→ 报告引用了某篇 Bloomberg 文章 → 搜索该文章

### 3.4 第四轮：补盲区

审视已搜集的材料：
→ 缺少买方视角？→ 搜 `{标的} fund manager view` / `{标的} 基金经理观点`
→ 缺少反面观点？→ 搜 `{标的} bear case` / `{标的} risk`
→ 缺少评论性文章？→ 搜 `{标的} Bloomberg` / `{标的} FT` / `{标的} Reuters`

---

## 第 4 步：去重与筛选

### 4.1 去重

- 同一机构的多篇报告 → 只保留最新/最深的一篇
- 转载/二手解读 → 丢弃，找原文
- 同一观点的不同表述 → 只保留论证最完整的

### 4.2 筛选标准

对每篇候选材料，问：

| 问题 | 不通过则降级或丢弃 |
|------|------------------|
| 有清晰的逻辑链条（不只是结论）？ | 纯观点/快讯 → 降级为"评论"，不作为核心材料 |
| 有一手数据支撑？ | 纯转引 → 降级 |
| 来源可信（机构/知名作者）？ | 匿名自媒体 → 丢弃 |
| 与用户问题严格相关？ | 只是沾边 → 丢弃 |

**不要用页数卡死。** 一篇 3 页的 Bloomberg 深度评论可能比一篇 20 页的水研报更有价值。看的是逻辑密度，不是长度。

---

## 第 5 步：分类与分层

将通过筛选的材料分为三层：

| 层级 | 标准 | 用途 |
|------|------|------|
| **核心材料** | 深度报告，完整论证链，一手数据 | 进入"读"步做金字塔拆解 |
| **补充评论** | 观点鲜明的短评/专栏，逻辑清晰但不够深 | 作为交叉验证的参考 |
| **数据源** | 权威机构的原始数据发布 | 进入"沉"步做 Source 注册 |

---

## 第 6 步：检查多样性

最终审视材料组合：

- **范式多样性**：是否覆盖了卖方（券商/投行）、买方（基金）、媒体（Bloomberg/FT）、独立研究？
- **视角多样性**：是否既有看多也有看空？既有基本面也有技术面/宏观面？
- **语言多样性**：国际性标的是否中英文都有？
- **深度层次性**：是否既有深度长报告，也有精炼短评？
- **时效性**：核心材料是否在最近 6-12 个月内？

→ 哪个维度缺了，回到第 3 步补搜。
